Introducción

¿Cómo le decimos a una computadora qué hacer?

Para resolver un problema con una computadora, el primer paso es comunicarle el algoritmo que resuelve el problema; por lo tanto para ser ejecutado deberá estar escrito en un lenguaje que la máquina conozca.

El conjunto de instrucciones que una máquina es capaz de ejecutar se denomina lenguaje de máquina y básicamente está formado por un conjunto de operaciones aritméticas, lógicas y de transferencia de datos, el cual es muy difícil de utilizar ya que está muy alejado de la forma en que una persona razona los algoritmos.

🗣️ Lenguaje Informal

Es la forma natural en que hablamos y escribimos. Es ambiguo y tiene ciertas carencias que dificultan la comunicación precisa con una máquina.

🤖 Lenguaje de Máquina

Conjunto de instrucciones que una máquina puede ejecutar directamente. Formado por operaciones aritméticas, lógicas y de transferencia de datos. Muy difícil de utilizar.

Existen herramientas utilizadas para traducir algoritmos al lenguaje de máquina, sin embargo ninguna es capaz de traducir el lenguaje informal que se viene utilizando; esto sucede debido a que el lenguaje informal es ambiguo y tiene ciertas carencias.

Si se le agregan reglas y nuevos elementos al lenguaje informal se obtiene un lenguaje formal el cual se denominará lenguaje de alto nivel o de programación. Por ser formal, este tipo de lenguaje sí es posible traducirlo a lenguaje de máquina.

Al proceso de traducción se le denomina compilación.

2

El Proceso de Compilación

Del algoritmo al programa ejecutable

Por lo tanto, escribiremos nuestros programas en un lenguaje de alto nivel. Luego le aplicaremos un compilador, el cual buscará errores de sintaxis y de semántica en nuestro programa de alto nivel.

En caso de encontrar alguno, nos informará del error. Debemos corregir dicho error y luego compilarlo nuevamente.

Cuando no quede ningún error de sintaxis ni de semántica, el compilador realizará la traducción a lenguaje de máquina. El programa compilado, ya en lenguaje de máquina, es el que llamaremos programa ejecutable y que el computador será finalmente capaz de ejecutar.

Bloques e Instrucción Compuesta

En C, a una secuencia de instrucciones encerrada entre llaves { } se le denomina bloque o instrucción compuesta.

Una instrucción compuesta puede contener tanto instrucciones simples como otras instrucciones compuestas.

Bloques y Estructuras de Control

Cómo controlar el flujo de ejecución de un programa

A continuación se presentan las estructuras de control vistas anteriormente y cómo se escriben en el lenguaje C. Las secuencias de pasos vistas previamente se corresponden con instrucciones de C. En caso de contener más de una instrucción, en el lenguaje C corresponde a una instrucción compuesta. O sea, deben escribirse dichas instrucciones encerradas entre llaves.

Condicional: si-entonces-sino

📝 Pseudocódigo
si condicion entonces secuencia_1 sino secuencia_2
💻 Lenguaje C
if (condicion) { instruccion_1; } else { instruccion_2; }

Bucle: repetir-mientras

📝 Pseudocódigo
repetir secuencia mientras condicion sea verdadera
💻 Lenguaje C
do { instruccion; } while (condicion);

Bucle: mientras-hacer

📝 Pseudocódigo
mientras condicion sea verdadera hacer secuencia
💻 Lenguaje C
while (condicion) { instruccion; }

Bucle: para-cada

📝 Pseudocódigo
para cada elemento de un conjunto hacer secuencia
💻 Lenguaje C
for (exp1; exp2; exp3) { instruccion; }

Selección múltiple: según-sea

📝 Pseudocódigo
según expresion sea valor_1: secuencia_1 …. otro: secuencia_N+1
💻 Lenguaje C
switch (exp) { case valor1: instruccion_1; break; …. default: instruccion_N; }

Identificadores

Las reglas para nombrar elementos en C

Los identificadores válidos en C deben cumplir las siguientes restricciones:

  • Secuencia de letras, dígitos o guión bajo (_)
  • Deben comenzar con una letra
  • Solo los primeros 31 caracteres son significativos
  • No pueden coincidir con palabras reservadas (if, switch, etc.)
  • Se distinguen mayúsculas de minúsculas (case-sensitive)

Variables y Constantes

Los objetos básicos de un programa

Las variables y constantes son los objetos básicos de un programa; pueden representar valores de diversos tipos, desde un simple número natural hasta el conjunto de empleados de una empresa.

Todos los datos que se quieran procesar deben estar almacenados de alguna forma en la memoria de la computadora. En los lenguajes de alto nivel no se manipula directamente la memoria, sino que se hace a través de variables y constantes.

Durante la ejecución del programa, estos objetos quedan almacenados en una cierta dirección de memoria ocupando un espacio que dependerá del tipo de dato de cada uno.

Tipos de Datos Básicos en C

En C, uno de los tipos que definen valores enteros se llama int. Otro es el tipo float, que permite representar valores reales o de punto flotante. Los valores 12.0 y 12 se consideran de distintos tipos.

El tipo de los caracteres en C se llama char. Los nombres de estos tipos, junto con los de otros tipos predefinidos de C son palabras reservadas, así como los nombres de las estructuras de control (if, switch, for, etc.)

🔢

int

Valores enteros

tipo_int.c
// Variable de tipo entero (int) int n = -560;
💯

float

Valores reales (punto flotante)

tipo_float.c
// Variable de tipo real (float) float f = 1714.326;
🔤

char

Caracteres individuales

tipo_char.c
// Variable de tipo carácter (char) char c = 'a';

Definición Formal de Variable

Una variable es una cuádrupla (N, V, T, D), donde:

  • N — Es el identificador que define el nombre de la variable
  • V — Es el valor que contiene (el cual puede variar)
  • T — Es el tipo de datos correspondiente a la variable
  • D — Es la dirección que ocupa en la memoria de la máquina

Declaración de Variables

Avisar al compilador para reservar memoria

Es necesario avisar al compilador que se utilizará una variable dentro de un programa, de forma de que se reserve la memoria necesaria para almacenar su contenido. El omitir un aviso tal, causará un error.

La forma en que se avisa el posterior uso de una variable, es declarándola previamente. Para declarar una variable, se debe especificar solamente el tipo y el nombre.

La dirección de memoria la otorga luego el compilador y es de bajo nivel. En nuestro programa en alto nivel, no interesa ni aporta conocer la dirección de memoria.

Sintaxis de Declaración

sintaxis_declaracion.c
// Sintaxis general para declarar variables id_tipo lista_ids_variables;

Ejemplos

ejemplos_declaracion.c
// Declaración de una variable entera int cantidad; // Declaración de dos variables reales en una sola línea float resultado, x; // Declaración de tres variables de tipo carácter char letra_1, letra_2, letra_3;

En el caso de que se quiera declarar más de una variable del mismo tipo, es posible declararlas todas de una vez, separando sus nombres con comas.

Se acostumbra (y de hecho, es considerado una buena práctica de programación) declarar todas las variables al comienzo del programa (más adelante veremos cómo).

Constantes Simbólicas

Valores con nombre que no cambian durante la ejecución

C permite la definición de constantes simbólicas, al permitir asociar un nombre simbólico a un valor de un tipo dado.

El propósito de utilizar constantes simbólicas en lugar de constantes de tipo es que para modificar el valor de una constante simbólica, únicamente se modifica en su definición, en lugar de modificar cada ocurrencia en el código del programa.

Por ejemplo, si queremos utilizar el valor del IVA en muchas expresiones dentro del programa, en lugar de escribir dicho valor en cada una, lo que hacemos es definirlo como constante simbólica. Luego, en cada expresión que haga uso del IVA, en lugar de colocar directamente el valor, lo que haremos es hacer uso de la constante simbólica.

Sintaxis

sintaxis_constante.c
// Sintaxis para declarar una constante simbólica const id_tipo ID_CONSTANTE = valor;

La definición de una constante simbólica es igual a la de una variable agregando el calificador const delante, y dándole un valor a la vez. Este valor no podrá ser cambiado durante la ejecución.

Se acostumbra, aunque no es obligatorio desde el punto de vista de la sintaxis del lenguaje, que los nombres de las constantes simbólicas se escriban siempre en MAYÚSCULAS.

También es una buena práctica de programación que las constantes simbólicas sean lo primero que se declare dentro del programa.

Ejemplos

ejemplos_constantes.c
// Constante simbólica de tipo char const char PRIMER_CAR = 'a'; // Constante simbólica de tipo int (valor del IVA) const int IVA = 23;

❌ Sin constante simbólica

Si el IVA cambia de 23 a 21, hay que buscar y modificar cada aparición del valor 23 en todo el código del programa.

✅ Con constante simbólica

Solo se modifica una vez en la definición de la constante. Todos los usos de la constante se actualizan automáticamente.

Programa

Algoritmo escrito en un lenguaje de programación de alto nivel

Un programa es un algoritmo escrito en un lenguaje de programación de alto nivel.

Los datos de entrada serán ingresados por el usuario a través del teclado. La salida será emitida al usuario a través de mensajes por pantalla y el método del algoritmo estará dado por las instrucciones (simples y/o compuestas) que contenga el programa.

Mirando al futuro: En futuros cursos de programación se verá también cómo hacer que un programa contenga más de un algoritmo en lugar de solamente uno. Dichos algoritmos se llamarán funciones y procedimientos y serán estudiados oportunamente. También se verán otros mecanismos para leer datos de la entrada y para emitir al usuario la salida. Para este curso, nos bastará con leer desde teclado y mostrar valores y texto en la salida estándar (consola).

Estructura Básica de un Programa en C

La anatomía que todo programa debe respetar

Se presenta a continuación la estructura que tendrán nuestros primeros programas en lenguaje C. Todo programa que escribamos debe respetar la siguiente estructura sintáctica:

estructura_programa.c
// ============================================ // Estructura básica de un programa en C // ============================================ // 1. Inclusión de librerías predefinidas #include <stdio.h> // 2. Declaración de constantes simbólicas const int IVA = 23; // 3. Programa principal int main() { // 3a. Declaración de variables int cantidad; float precio, total; // 3b. Instrucciones (tanto simples como compuestas) // ... aquí va el código del programa ... return 0; }

Conforme vayamos avanzando en el curso, iremos incorporando nuevos elementos sintácticos a la estructura básica de un programa. Por ahora, nos bastará con la estructura recién definida.

Detalle de cada sección

📦 1. Inclusión de Librerías Predefinidas

▼

Lo primero que debe haber en el programa es la inclusión de librerías predefinidas. Veremos que existen muchas librerías predefinidas, que permiten hacer uso de diferentes elementos ya predefinidos del lenguaje.

La primera librería que usaremos se llama <stdio.h> y es la que nos permitirá hacer uso de las instrucciones necesarias para poder leer valores por teclado y emitir al usuario tanto texto como valores por pantalla.

🔒 2. Declaración de Constantes Simbólicas

▼

A continuación de la inclusión de la(s) librería(s) predefinidas que sean necesarias, debe ir la declaración de las constantes simbólicas que se vayan a utilizar en el programa.

Cada una de ellas se debe declarar conforme a la sintaxis vista anteriormente para las constantes simbólicas.

🎯 3. Programa Principal (main)

▼

Luego de las constantes simbólicas, viene otra sección que denominaremos programa principal (también llamado bloque principal) de nuestro programa.

Dentro de dicha sección se coloca la declaración de las variables, así como las instrucciones (tanto simples como compuestas) que el programa vaya a utilizar.

El programa o bloque principal tiene el siguiente encabezado: int main() y debe escribirse exactamente así. En futuros cursos de programación se verá por qué es que se escribe de esa manera. Por ahora, nos alcanza con saber que dicha línea, así escrita, marca el comienzo del bloque principal del programa.

Luego de la línea conteniendo el encabezado del bloque principal (main), se colocan llaves. Dentro de dichas llaves es que deben ir todas las declaraciones de las variables que el programa utilice, así como todas las instrucciones (tanto simples como compuestas) que el programa vaya a utilizar.

Buenas Prácticas de Programación

Escribir código claro, legible, ordenado y prolijo

Observación: Si bien la sintaxis de C permite alternar la declaración de las variables junto con las instrucciones dentro del bloque main, se considera una buena práctica de programación que primero se declaren juntas todas las variables que el programa vaya a utilizar y que recién luego de ellas es que se coloquen las instrucciones a ejecutar.

El compilador admitirá alternar ambos elementos sin problema. Es decir, desde el punto de vista sintáctico, ello es posible. No obstante, conforme a las buenas prácticas de programación, nosotros siempre declararemos primero las variables a utilizar y colocaremos luego las instrucciones a ejecutar.

A medida que avancemos en el curso, veremos que no solamente es importante que nuestros programas sean sintácticamente correctos y que compilen correctamente.

También importa que se sigan buenas prácticas de programación a la hora de escribir nuestros programas. Esto ayuda a que los programas sean:

Entrada y Salida Estándar

Comunicación entre el usuario y el programa

Como se dijo anteriormente, los datos de entrada del programa serán ingresados por el usuario a través de teclado y los datos de salida del programa (tanto texto como valores) serán mostrados a través de la consola.

Se suele llamar Entrada Estándar a los datos que se leen por teclado y Salida Estándar a los valores y mensajes mostrados a través de la consola.

Entrada/Salida en C

En el lenguaje C, la forma de leer o escribir en los dispositivos de entrada o salida estándar está dada por un par de operaciones: scanf y printf.

Estas operaciones se proveen en una librería predefinida denominada <stdio.h>, por tal razón, para el uso de las mismas es necesario incluir al comienzo del programa una línea especial que tiene dicho cometido:

stdio_include.c
// Inclusión obligatoria para usar scanf y printf #include <stdio.h>
14

scanf — Lectura de Datos

Ingreso de datos desde la entrada estándar hacia el programa

Esta operación permite el ingreso de datos desde la entrada estándar hacia el programa.

Sintaxis

sintaxis_scanf.c
// Sintaxis de scanf para leer datos desde el teclado scanf("cadena_de_formato", &param1, &param2, ... , &paramN);

El primer parámetro (en este caso cadena) es una cadena de caracteres entre comillas dobles (") que indica el tipo de los valores que serán leídos y colocados en las variables que siguen a continuación (parámetros) que son identificadores de variables previamente definidas.

Importante: Una instrucción que invoca al procedimiento scanf tiene por efecto detener la ejecución del programa hasta que se ingresen tantos valores como parámetros se hayan especificado, los que se almacenarán en la entrada estándar en el orden que llegaron. Estos valores se tomarán de allí, de a uno por vez, empezando por el primero, y se asignarán a las variables de acuerdo al orden en que estas aparecen en la lista.

Algunas Invocaciones a scanf

ejemplo_scanf_entero.c
// Ejemplo 1: Leer un valor entero scanf("%d", &cantidad); // La cadena "%d" indica que se lee un entero (int) // El valor ingresado se almacena en la variable 'cantidad'
ejemplo_scanf_caracter.c
// Ejemplo 2: Leer un carácter scanf("%c", &letra); // La cadena "%c" indica que se lee un carácter (char) // El valor ingresado se almacena en la variable 'letra'
ejemplo_scanf_dos_reales.c
// Ejemplo 3: Leer dos valores reales (float) scanf("%f %f", &precio_nafta, &cant_litros); // La cadena "%f %f" indica que se leen dos números reales // El primer valor se almacena en 'precio_nafta' // El segundo valor se almacena en 'cant_litros'

El símbolo &: El símbolo & colocado delante de los identificadores de las variables, es necesario por razones que se verán en futuros cursos de programación. Por ahora, nos basta con saber que es parte de la sintaxis necesaria para que la instrucción scanf funcione correctamente.

Especificadores de Formato

Especificador Tipo de Dato Descripción
%d int Número entero
%f float Número real (punto flotante)
%c char Carácter individual

printf — Escritura de Datos

Escribir datos en la salida estándar

Con la operación printf se escriben datos en la salida estándar. El origen de éstos valores puede ser diverso: variables, constantes simbólicas, constantes de tipo o expresiones.

Sintaxis

sintaxis_printf.c
// Sintaxis de printf para escribir datos en la salida estándar printf("cadena_de_formato", param1, param2, ... , paramN);

El primer parámetro (en este caso cadena) es una cadena de caracteres como en scanf que indica el tipo de los parámetros que se quieren escribir y que corresponden a los identificadores de variables, constantes o expresiones que siguen en la lista.

Diferencia con scanf: A diferencia de scanf, en printf también es posible escribir un texto que se desee que salga en pantalla, intercalándolo con los símbolos para indicar el tipo de los valores a mostrar.

El efecto de una instrucción que invoca a printf es escribir en la salida estándar los valores de los parámetros, en el orden en que se especificaron.

Ejemplos

ejemplo_printf_real.c
// Ejemplo 1: Mostrar un valor real (float) printf("%f", precio); // Muestra por pantalla el valor almacenado en 'precio'

Se indica que se va a mostrar un valor de tipo real, y a continuación se coloca la variable precio (de tipo float). El efecto en la ejecución es que se muestra por pantalla el valor almacenado dentro de la variable precio.

ejemplo_printf_caracter.c
// Ejemplo 2: Mostrar un carácter (char) printf("%c", letra); // Muestra por pantalla el valor almacenado en 'letra'

Se indica que se va a mostrar un valor de tipo carácter, y a continuación se coloca la variable letra (de tipo char). El efecto en la ejecución es que se muestra por pantalla el valor almacenado dentro de la variable letra.

ejemplo_printf_texto_intercalado.c
// Ejemplo 3: Intercalar texto con valores printf("Hay %d litros", cant_litros); // Si cant_litros = 10, la salida será: "Hay 10 litros"

Se intercala texto con un valor de tipo entero, y a continuación se coloca la variable cant_litros (de tipo float).

El efecto en la ejecución es que se muestra por pantalla un mensaje formado por la palabra Hay, seguido del valor almacenado en la variable cant_litros y seguido luego de la palabra litros.

Por ejemplo, si la variable cant_litros contiene el valor 10, el mensaje que verá el usuario es:

Hay 10 litros
ejemplo_printf_texto_solon.c
// Ejemplo 4: Solo texto con salto de línea (\n) printf("Esto es solamente un mensaje\n"); // \n = salto de línea: el próximo printf aparecerá en la línea siguiente

Solamente se muestra texto al usuario, sin incorporar el valor de ninguna variable. Para ello, simplemente se coloca el texto a mostrar entre comillas dobles, como único parámetro de printf.

Esto genera que se muestre al usuario el texto contenido entre las comillas. El carácter especial \n al final significa salto de línea.

Esto quiere decir que, tras mostrar el mensaje, el programa dará un salto de línea en la pantalla. Por lo tanto, cualquier otro mensaje que se muestre posteriormente, aparecerá en la línea siguiente.

En caso de que no se coloque el carácter de salto de línea, lo próximo que se despliegue aparecerá en la misma línea, justo a continuación de la palabra mensaje.

Recuerda: \n = salto de línea. Sin él, todo aparece en la misma línea. Con él, cada mensaje aparece en una línea nueva.
